# RadishMD 开发者手册

## 简介

本手册面向 RadishMD 项目的开发者和贡献者，介绍项目开发环境搭建、代码结构、打包部署等相关内容。

---

## 环境要求

### 通用要求

- Node.js 18+
- Rust 1.70+
- npm 或 pnpm

### Windows 开发

- Windows 10/11
- Visual Studio Build Tools (MSVC)
- Rust MSVC 目标

### Linux 开发

- Ubuntu 20.04+ 或其他主流 Linux 发行版
- build-essential
- libwebkit2gtk-4.1-dev
- libssl-dev

### macOS 开发

- macOS 10.15+
- Xcode Command Line Tools
- Rust macOS 目标

## 项目结构

```
/src
  App.tsx                    # 主应用编排层
  hooks/
    useEditorWorkspace.ts    # 工作区状态抽象
  main.tsx                   # React 入口
  index.css                  # Tailwind CSS + 主题变量
  /components
    MilkdownEditor.tsx      # Milkdown 编辑器组装层
    Toolbar.tsx              # 工具栏薄入口
    milkdown/               # 编辑器能力拆分模块
    toolbar/                # 工具栏控制层、视图层、配置层
    /ui                      # shadcn-ui 组件
      button.tsx
      separator.tsx
      dropdown-menu.tsx
      dialog.tsx
  /lib
    utils.ts                 # 工具函数
/src-tauri
  src/main.rs               # Rust 入口
  src/lib.rs                # 插件初始化
  Cargo.toml                # Rust 依赖
  tauri.conf.json           # Tauri 配置
  capabilities/             # 插件权限配置
/public
  title.svg                # 应用图标
```

## 当前结构优化方案

本项目当前采用“页面编排层 + 工作区状态层 + 编辑器层 + 工具栏控制/视图/配置层”的拆分方式，目标是让后续团队协作更容易维护和扩展。

### 1. 页面编排层

路径：`src/App.tsx`

职责：
- 只负责组合编辑器和工具栏
- 通过 `useEditorWorkspace` 管理页面级状态
- 不再直接承载文件操作和工具栏行为

### 2. 工作区状态层

路径：`src/hooks/useEditorWorkspace.ts`

职责：
- 管理 `content`、`theme`、`currentFilePath`、`editorKey`
- 管理 `editorActionsRef`
- 统一处理打开文件后的状态刷新

### 3. 编辑器层

路径：`src/components/MilkdownEditor.tsx`

职责：
- 负责 Milkdown 初始化与命令暴露
- 负责编辑器主题挂载
- 负责数学块、代码块语言选择器等扩展能力组装

拆分子模块：
- `src/components/milkdown/types.ts`
- `src/components/milkdown/mathBlock.ts`
- `src/components/milkdown/useCodeBlockLanguageSelector.ts`
- `src/components/milkdown/CodeBlockLanguageSelector.tsx`
- `src/components/milkdown/codeBlockLanguages.ts`

### 4. 工具栏层

路径：`src/components/toolbar/`

职责：
- `useToolbarController.ts` 负责文件操作、编辑动作、快捷键监听、UI 状态
- `ToolbarMenus.tsx` 负责菜单展示和动作分发
- `ToolbarDialogs.tsx` 负责关于、快捷键、表格对话框
- `toolbarConfig.ts` 负责关于页文案和快捷键分组
- `toolbarMenuConfig.ts` 负责菜单结构、分组、快捷键提示和动作映射

### 5. 设计原则

- 单一职责：一个模块只负责一类问题
- 分层解耦：状态、动作、视图、静态配置分开
- 配置驱动：菜单、快捷键、文案尽量由配置提供
- 保持兼容：外部调用方式尽量不变，避免改动扩散
- 可持续演进：新功能优先新增模块，不优先扩写大文件

### 6. 后续推进建议

- 将工具栏菜单进一步数据化，统一生成菜单 UI
- 为 Toolbar 动作增加更明确的 action map 层
- 将工作区状态继续抽象为独立 store，方便接入最近文件、历史记录、持久化
- 为关键动作补充单元测试或轻量集成测试

## 开发环境搭建

### 1. 克隆项目

```bash
git clone <repository-url>
cd RadishMD
```

### 2. 安装前端依赖

```bash
npm install
```

### 3. 配置 Rust 环境

#### Windows

```bash
# 安装 Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 添加 MSVC 目标
rustup target add x86_64-pc-windows-msvc
```

#### Linux

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install build-essential libwebkit2gtk-4.1-dev libssl-dev

# 安装 Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

#### macOS

```bash
# 安装 Xcode Command Line Tools
xcode-select --install

# 安装 Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

### 4. 运行开发模式

#### Windows

```bash
powershell.exe -ExecutionPolicy Bypass -File run_dev.ps1
```

或直接运行：

```bash
npm run tauri dev
```

#### Linux / macOS

```bash
npm run tauri dev
```

首次运行需要编译 Rust 代码，可能需要 10-20 分钟。

## 开发协作说明

当前项目结构已经从单文件堆叠转向分层架构。开发时建议优先遵循以下边界：

- 页面级状态优先放在 `src/hooks/useEditorWorkspace.ts`
- 编辑器能力优先拆到 `src/components/milkdown/`
- 工具栏动作、视图、静态配置优先拆到 `src/components/toolbar/`
- 新菜单项优先改配置，不直接在 JSX 中堆叠逻辑
- 新快捷键优先同步到 `toolbarConfig.ts` 和 `toolbarMenuConfig.ts`

## 版本同步说明

当前项目的版本号采用“单一来源 + 自动同步”的方式管理。

### 版本源头

只需要修改根目录 `package.json` 中的 `version` 字段。

同步脚本会自动把版本号写入以下文件：

- `package-lock.json`
- `src-tauri/tauri.conf.json`
- `src-tauri/Cargo.toml`

### 可用命令

```bash
npm run version:sync
npm run version:check
```

- `npm run version:sync`：手动同步版本号
- `npm run version:check`：检查版本是否漂移

### 自动同步时机

以下命令在执行前会自动同步版本号：

- `npm run build`
- `npm run preview`
- `npm run tauri`
- `npm run dev`

如果你只是想升级版本，推荐的操作顺序是：

1. 修改 `package.json` 的 `version`
2. 运行 `npm run version:check` 或直接执行 `npm run build`
3. 提交变更

## 构建与打包

### 构建前端

```bash
npm run build
```

说明：构建前会先执行版本同步，所以不需要再手动修改 `tauri.conf.json` 或 `Cargo.toml`。

### 构建 Tauri 应用

```bash
npm run tauri build
```

说明：`npm run tauri build` 之前同样会先同步版本号，确保前端、Tauri 配置和 Rust 包版本一致。

构建完成后，安装包位于：

- Windows: `src-tauri/target/release/bundle/nsis/`
- Linux: `src-tauri/target/release/bundle/deb/` 或 `AppImage`
- macOS: `src-tauri/target/release/bundle/dmg/`

---

## Windows 打包

### 方法一：使用 npm 命令

```bash
npm run tauri build
```

这将生成：

- `RadishMD_x.x.x_x64-setup.exe` (NSIS 安装程序)
- `RadishMD_x.x.x_x64.msi` (MSI 安装包，可选)

### 方法二：单独构建

```bash
# 进入 Tauri 目录
cd src-tauri

# 构建发布版本
cargo build --release

# 打包成安装程序
cargo tauri build
```

### Windows 打包注意事项

1. 确保已安装 Visual Studio Build Tools
2. 需要添加 MSVC 目标：`rustup target add x86_64-pc-windows-msvc`
3. 图标文件位于 `src-tauri/icons/`
4. 配置文件位于 `tauri.conf.json`

## Linux 打包

### 安装 Linux 构建依赖

#### Ubuntu/Debian

```bash
sudo apt update
sudo apt install build-essential libwebkit2gtk-4.1-dev libssl-dev curl wget file
```

#### Fedora

```bash
sudo dnf install gcc gcc-c++ make openssl-devel webkit2gtk4.1-devel
```

#### Arch Linux

```bash
sudo pacman -S base-devel openssl webkit2gtk-4.1
```

### 构建 Linux 应用

```bash
npm run tauri build -- --target x86_64-unknown-linux-gnu
```

### 打包格式

#### Debian/Ubuntu (.deb)

```bash
cargo tauri build --bundles deb
```

#### AppImage

```bash
cargo tauri build --bundles appimage
```

#### RPM

```bash
cargo tauri build --bundles rpm
```

### Linux 打包注意事项

1. WebView2 依赖：大多数现代 Linux 发行版已预装
2. 确保在 Docker 或虚拟机中交叉编译，以生成兼容更多发行版的二进制文件
3. 使用 `tauri.conf.json` 中的 `bundle.linux.deb` 配置自定义包信息

## macOS 打包

### 环境要求

- macOS 10.15+
- Xcode 15+
- Apple Developer 证书（用于签名）

### 构建 macOS 应用

```bash
npm run tauri build
```

### 打包格式

#### .dmg (磁盘镜像)

```bash
cargo tauri build --bundles dmg
```

#### .app (应用Bundle)

```bash
cargo tauri build --bundles app
```

### 代码签名与公证

#### 开发证书签名

```bash
# 签名应用
codesign --sign "Developer ID Application: Your Name" --timestamp "http://timestamp.apple.com/ts01" --options runtime ./path/to/app

# 验证签名
codesign --verify --verbose=2 ./path/to/app
```

#### App Store 分发

```bash
# 需要 Apple ID 和 App Store Connect 证书
cargo tauri build --bundles app
```

#### 公证 (Notarization)

```bash
# 上传进行公证
xcrun notarytool submit ./path/to/app.dmg --apple-id "your@email.com" --password "app-specific-password" --team-id "YOUR_TEAM_ID"

# 等待公证完成
xcrun notarytool wait ./path/to/app.dmg --apple-id "your@email.com" --password "app-specific-password" --team-id "YOUR_TEAM_ID"
```

### macOS 打包注意事项

1. 需要 Apple Developer 账户才能进行代码签名和公证
2. 图标需要 .icns 格式
3. 使用 `tauri.conf.json` 中的 `bundle.macOS` 配置
4. Rust 目标：`rustup target add x86_64-apple-darwin aarch64-apple-darwin`

## 跨平台构建

### 使用 Docker 进行 Linux 交叉编译

创建 Dockerfile：

```dockerfile
FROM ubuntu:22.04

RUN apt update && apt install -y \
    build-essential \
    curl \
    wget \
    file \
    libwebkit2gtk-4.1-dev \
    libssl-dev \
    pkg-config \
    libayatana-appindicator3-dev

RUN curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
ENV PATH="/root/.cargo/bin:${PATH}"

WORKDIR /app
COPY . .
RUN npm install && npm run build
RUN cargo build --release --manifest-path src-tauri/Cargo.toml --target x86_64-unknown-linux-gnu
```

构建：

```bash
docker build -t radishmd-builder .
docker run --rm -v $(pwd):/app/output radishmd-builder \
    cp src-tauri/target/x86_64-unknown-linux-gnu/release/radishmd /app/output/
```

## 配置说明

### tauri.conf.json

主要配置项：

```json
{
  "productName": "RadishMD",
  "version": "0.1.0",
  "identifier": "com.radishmd.app",
  "app": {
    "windows": [{
      "title": "RadishMD",
      "width": 800,
      "height": 600
    }]
  },
  "bundle": {
    "active": true,
    "targets": "all",
    "icon": [
      "icons/32x32.png",
      "icons/128x128.png",
      "icons/128x128@2x.png",
      "icons/icon.icns",
      "icons/icon.ico"
    ]
  }
}
```

其中 `version` 字段不建议手动维护，实际应由根目录 `package.json` 的版本号同步生成。

### 更换图标

1. 准备 SVG 或 PNG 图标（建议 512x512 以上）
2. 运行图标生成命令：

```bash
npx tauri icon public/your-icon.svg -o src-tauri/icons
```

支持的输入格式：SVG、PNG

## 常见问题

### Q: Windows 构建失败？

1. 检查 Rust 和 MSVC 目标是否正确安装
2. 确保 Visual Studio Build Tools 已安装

### Q: Linux 构建失败？

1. 检查系统依赖是否完整安装
2. 尝试更新 Rust：`rustup update`

### Q: macOS 签名失败？

1. 检查证书是否有效
2. 确保钥匙串访问已授权

### Q: 应用图标不显示？

1. 重新生成图标：`npx tauri icon public/title.svg`
2. 清除缓存并重新构建
